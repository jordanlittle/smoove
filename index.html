Tables

In order to retain a truly responsive experience on mobile, tables must be wrapped in a div.table-scroll. This allows mobile browsers to view wider tables by enabling touch-scroll. Using the table class keeps default tables easily selectable.

Use colspan-[1-12] classes to format table cells in a 12 column grid.

<div class="table-scroll">
	<table class="table">
		<!-- content -->
	</table>
</div>

Utilities

These are small helper classes that usually only do one thing.

Add a class of text-left for left-aligned text.

Add a class of text-center for centered text.

Add a class of text-right for right-aligned text.

Use a class of no-layout to kill any padding/margin on elements.

To hide an element on all screens, add a class of hidden.

To hide an element only on smaller screens, a class of hidden-on-mobile does the trick.

In order to only show an element on smaller screens, apply a class of show-only-on-mobile.

Use pad-top, pad-bottom, pad-left, and pad-right to get a device-responsive amount of padding in the requested direction. This element uses pad-left: Hey there.

Cards

A very generic class meant to be used for highlighting a certain section with a white background and a subtle box-shadow. Adding a class of card applies the background and shadows. Include a class of card-pad-[size] (using the sizes lg, md, and sm) to add padding of various sizes.

Avoid mixing the grid classes with these card classes. Use the grid to create a scaffold around cards.

Card-pad sizes are used on cards to keep a separate scope from the global pad-[direction] classes.

<div class="u-md-6 u-sm-6">						
	<div class="card  card-pad-md">
		<!-- ... -->
	</div>
</div>

Dropdowns

Dropdowns must be wrapped in a containing element with a class of .has-dropdown. There are two pieces of a dropdown once inside this container: the trigger and the target. The trigger is defined with the data-attribute data-ui-trigger, while the target is defined with data-ui-target. The trigger must also have a data-dropdown attribute which will contain the HTML id of the target element.

<div class="has-dropdown">
    <a href="#" data-dropdown="dd_1" data-ui-trigger>Activate!</a>
    <div class="dropdown" id="dd_1" data-ui-target>
        <!-- ... -->
    </div>
</div>

Grid System

Smoove uses a 12-column grid. Each row begins with a class of "g" and contains units of class "u-[size]-[n]" â€“ md targets larger screens, sm for smaller.

<!-- Start a row of grids with class "g" -->
<div class="g">
<!--
Columns are defined as units starting with a class of "u-".
The modifier '-md' means 'medium screens and above'.
The modifier '-sm' means 'small screens only'.
There are twelve columns in -md and six columns in -sm'.
If you wanted a 10/12 on desktop and a 6/12 on mobile:
-->
    <div class="u-md-10 u-sm-6">
    </div>
</div>

Nesting

Grid units can be nested. Once you're inside a "g" class, start a new row (and thus a new 12-col grid context) by nesting another <div class="g">...</div>.

Offsets

You can use offset-[size]-[n] classes to move units to the right in the grid.

<div class="g">
<!--
    Use "offset-" classes to move columns horizontally.
    "offset-md-" applies to larger screens.
    "offset-sm-" applies to smaller ones.
    Ensure your units and offsets add up to 12.
-->
    <div class="u-md-3  u-sm-3">
    </div>
    <div class="u-md-3  u-sm-3">
    </div>
    <div class="u-md-3 offset-md-3  u-sm-3 offset-sm-3">
    </div>
</div>

Push & Pull

You can use push-[size]-[n] and pull-[size]-[n] classes to move units around the grid irrespective of their source order. Push moves a column to the right while Pull moves it to the left.

<div class="g">
<!--
    Use "push" classes to move columns right, regardless of source order.
    Use "pull" classes to move columns left, regardless of source order.
-->
    <div class="u-md-3  push-md-9">
        <!-- I should be on the far left, but I'm on the far right -->
    </div>
    <div class="u-md-6">
        <!-- I'll stay in the center -->
    </div>
    <div class="u-md-3  pull-md-9">
        <!-- I should be on the far right, but I'm on the far left -->
    </div>
</div>
