Tables

In order to retain a truly responsive experience on mobile, tables must be wrapped in a div.table-scroll. This allows mobile browsers to view wider tables by enabling touch-scroll. Using the table class keeps default tables easily selectable.

Use colspan-[1-12] classes to format table cells in a 12 column grid.


<div class="table-scroll">
	<table class="table" title="Sunday">
		<input name="day[]" type="checkbox" id="sun" class="day-picker-input" />

	</div>

Grids

Smoove uses a 12-column grid. Each row begins with a class of "g" and contains units of class "u-[size]-[n]" – md targets larger screens, sm for smaller.


<!-- Start a row of grids with class "g" -->
<div class="g">
<!--
Columns are defined as units starting with a class of "u-".
The modifier '-md' means 'medium screens and above'.
The modifier '-sm' means 'small screens only'.
There are twelve columns in -md and six columns in -sm'.
If you wanted a 10/12 on desktop and a 6/12 on mobile:
-->
	<div class="u-md-10 u-sm-6">
	</div>
</div>

Grid units can be nested. Once you're inside a "g" class, start a new row (and thus a new 12-col grid context) by nesting another <div class="g">...</div>.

You can use offset-[size]-[n] classes to move units to the right in the grid.


<div class="g">
<!--
Use "offset-" classes to move columns horizontally.
"offset-md-" applies to larger screens.
"offset-sm-" applies to smaller ones.
Ensure your units and offsets add up to 12.
-->
	<div class="u-md-3  u-sm-3">
	</div>
	<div class="u-md-3  u-sm-3">
	</div>
	<div class="u-md-3 offset-md-3  u-sm-3 offset-sm-3">
	</div>
</div>

Add a class of g-gutters to get a guttered grid. Guttered grids will lose their gutters on mobile to save space.


<div class="g  g-gutters">
<!--
Add the class "g-gutters" to your .g class for guttered grids.
The gutters are fixed and are set in _build.scss ($grid-gutter-size).
-->
	<div class="u-md-3">
	</div>
	<div class="u-md-3">
	</div>
	<div class="u-md-3">
	</div>
	<div class="u-md-3">
	</div>
</div>

Smoove Cards & Card Padding

Smoove features the idea of "cards." A very generic class meant to be used for highlighting a certain section with a white background and a subtle box-shadow. Adding a class of card applies the background and shadows. Include a class of card-pad-[size] (using the sizes lg, md, and sm) to add padding of various sizes.

Avoid mixing the grid classes with these card classes. Use the grid to create a scaffold around cards.

Card-pad sizes are used on cards to keep a separate scope from the global pad-[direction] classes.


<div class="u-md-6 u-sm-6">						
	<div class="card  card-pad-md">
		<!-- ... -->
	</div>
</div>

Smoove Tabs

Any range of elements can be used to create tabs. A simple <ul> tag could be used, as long as an <a href="#" /> tag is used for the actual link. A tab group is linked by the data-attribute: data-sb-tab-group. Each tab activator must have an href attribute specifying the ID of the tab to be activated.

Tabs resize based on their content and automatically on viewport resize. Pick from three nav sizes (sm,md,lg). Use data-sb-tab-transition for turning on and off transitions.


<nav class="sb-tabs--nav  sb-tabs--nav-[sm,md,lg]" data-sb-tab-group="my_tabs">						
	<a class="button--link  sb-tab" href="#tab_1">Tab 1</a>
	<a class="button--link  sb-tab" href="#tab_2">Tab 2</a>
</nav>
<div class="sb-tabs--panels" data-sb-tab-transition="true" data-sb-tab-group="my_tabs">						
	<div class="sb-tab--panel" id="tab_1">
		<!-- Panel 1 contents -->
	</div>
	<div class="sb-tab--panel" id="tab_2">
		<!-- Panel 2 contents -->
	</div>
</div>

Before you can begin to determine what the composition of a particular paragraph will be, you must first decide on an argument and a working thesis statement for your paper. What is the most important idea that you are trying to convey to your reader? The information in each paragraph must be related to that idea. In other words, your paragraphs should remind your reader that there is a recurrent relationship between your thesis and the information in each paragraph. A working thesis functions like a seed from which your paper, and your ideas, will grow. The whole process is an organic one—a natural progression from a seed to a full-blown paper where there are direct, familial relationships between all of the ideas in the paper.
